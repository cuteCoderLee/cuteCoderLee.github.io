Linux

基本知识

硬件 内核（kernel） 外壳（shell：用户与内核间的命令解释器） 应用程序
sh csh ksh
特点：

  • 多任务（cpu时间分片分给不同进程)
  • 多用户（同时)
  • 管道（前一个程序的输出作为后一个程序的输入）
  • shell解释型高级语言，强大
  • 安全保护机制
  • 稳定
  • 用户界面：常用命令行界面，提供图形界面
  • 网络支持：TCP/IP协议
  • 移植性好

    用户和用户组管理

    UID：/etc/passwd（7个字段 root：x:0:0:root:/root:/bin/bash)
    GID：/etc/group (root:x:0:)
    用户查询： id finger（用户属性）
    新增用户： useradd[参数][用户名]
    设置密码：passwd
    修改属性：usermod[用户名] -d -g
    新增用户组：groupadd
    删除用户组：groupdel
    修改用户组：groupmod[-g -n][用户组名]

    文件和目录管理

    绝对路径： /
    相对路径：不一/开始 .当前目录 ..上一级目录
    显示当前目录：pwd
    变更： cd 没有任何参数，回到当前用户的家目录
    新增：mkdir[-m 存取模式（掩码）][-p建立其所有不存在的父目录] dirname
    删除：rmdir[-p] dirname 必须空目录,对其父目录具有写权限
    复制：cp源文件 目标
    移动：mv
    删除：rm[-ir]文件或目录提示
    查找：find[路径][参数] -name -user
    查看文件内容：cat（直接查看不翻页） more（翻页） less head tail

    文件系统管理

    存储和组织计算机数据的方法
    传统型文件系统：无日志，先写入数据，再写入元数据；ext2
    日志型文件系统：先写入日志记录文件；ext3 ReiserFS
    索引式：文件属性与内容存放在不同区块
    非索引式：只有block；FAT（windows）

    挂载：将文件系统与目录树关联

LVM配置

逻辑卷管理 提供灵活的管理磁盘空间的方法
LVM v1不能在线加大分区（在线？）
LVM v2可以在线扩大分区，有弹性
由上向下：
LV 逻辑卷，从卷组划分的虚拟分区，并对其格式化
VG 卷组，一个或多个物理卷
PV 物理卷，标记为LVM的可用空间(/dev/sda5 0x8e分区 or /dev/sda 未分区 or /dev/md0 RAID?)
补充：RAID（Redundant Array of Independent Disks）磁盘阵列，增加资料整合度，增强容错，增加处理量或容量
分区类型：

  • 82：swap
  • 83：ext3/ext4
  • 8e：LVM分区
  • fd：RAID分区

管道与重定向

程序执行的输入输出流:
标准输入 STDIN 0：键盘默认
标准输出 STDOUT 1：终端默认
标准错误 STDERR 2：终端默认
重定向运算符：
> 将输出重定向到文件，覆盖文件内容
>> 将输出重定向到文件，追加文件内容
< 重定向输入，如将键盘输入改由读入文件提供
管道：
| 将一个命令的STDOUT发送到另一个命令的STDIN
tee T管道 将上一个命令的STDOUT通过T管道重定向到文件，再发送给另一个命令的STDIN
比较管道与重定向:

  • 标准命令：grep root /etc/passwd
  • 重定向：grep root</etc/passwd
  • 管道：cat etc/passwd | grep root
